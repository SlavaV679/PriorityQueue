Реализована потокобезопасная in-memory очередь с поддержкой приоритетов, таймаутов и отложенной обработки.

Работа с очередью осуществляется через следующие методы:

	1. void Enqueue(T item, int priority); // Добавляет сообщение в очередь. Чем меньше число priority, тем выше приоритет
	2. void Enqueue(T item, int priority, TimeSpan delay) // Добавляет сообщение в очередь, после истечения delay
	3. Task<T> DequeueAsync(CancellationToken cancellationToken = default); // выгружается сообщение. Если сообщений нет, тогда происходит ожидание сообщений в очередь.
	4. Task<T> DequeueAsync(TimeSpan timeout, CancellationToken cancellationToken = default); // Выгружается сообщение из очереди. Если сообщений в очереди нет, тогда происходит ожидание в течении timeout.
	5. bool TryDequeue(out T item); // производится попытка выгрузить сообщение. Если удалось выгрузить сообщение, тогда возвращается true и в item сохраняется сообщение.
	6. int Count { get; } // возвращается количество сообщений в очереди.

Написаны unit-тесты, покрывающие:
	1. Базовую работу (FIFO в рамках одного приоритета)
	2. Приоритетность (сообщение с приоритетом 0 обрабатывается раньше, чем с приоритетом 5)
	3. Потокобезопасность
	4. Отложенные сообщения (сообщения не доступны до истечения задержки)

В процессе реализации были задействованы следующие аспекты .Net:
	1. Многопоточность и синхронизация (lock, SemaphoreSlim, Channel, ConcurrentQueue).
	2. Асинхронное программирование.
	3. Приоритетность сообщений
	4. Тестирование многопоточного кода.
